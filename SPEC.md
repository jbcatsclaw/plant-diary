# Plant Diary（植物观察日记）— MVP SPEC

## 目标
为家庭种植小植物用户（女性为主）提供一个温柔治愈、低负担的植物管理与观察记录工具。

## MVP 关键约束
- 平台：iOS + Android（Expo）
- 语言：简体中文
- 数据：全部 mock（本地内存/静态 JSON），不接数据库
- AI：全部 mock（后续再接真实模型）
- Web：提供 Vercel 预览（用于验收 UI/交互）

## 页面（MVP）
1) 花园（Garden）
- 植物网格：照片/名称/品种（可选）
- 搜索（名称/品种）
- 添加/编辑/删除植物（表单）

2) 植物详情（Plant Detail）
- 资料卡：照片、名称、品种、到家日期（可选）、备注（可选）
- 记录时间轴：最新在最上
- 过滤：浇水/施肥/修剪/除虫/其他
- 新增记录：类型、备注、日期；其他类型可自定义文本

3) 植物助手（AI mock）
- 每株植物一个对话页
- 基于最近记录给：下一步建议/风险提示/追问

## UI 风格
- 温柔治愈、偏 Notion-like 干净但更温暖（米白背景、低饱和绿色点缀）
- 大按钮、触控友好、减少信息噪音
- 禁止 emoji（统一用文字/图标）

## 验收标准
- Web 预览可用（Vercel）
- 核心流程跑通：花园→新增植物→进入详情→新增记录→时间轴可见
- 无 runtime 崩溃（ErrorBoundary 兜底）

## 非目标
- 云同步/账号
- 真 AI 诊断
- 复杂任务/日历（后续迭代）
